## 2024-05-18 - Added Escape key dismissal and screen reader accessibility for shortcuts
**Learning:** When implementing global keyboard shortcuts (like `1-6` or `l`), UI visual hints (e.g., `<span class="shortcut">1</span>`) can create confusing and nonsensical text when read by screen readers (like "Enable location l"). Furthermore, Escape key functionality should respect a navigation history stack rather than blindly jumping to a static screen to prevent users from getting lost when navigating back from modals like the neighborhood picker.
**Action:** Always add `aria-hidden="true"` to visual keyboard shortcut spans so they are ignored by the accessibility tree. Implement an array-based history stack (e.g., `st.screenHistory`) to track state transitions, allowing the Escape key and explicit Back buttons to pop the stack and restore the previous specific view.
