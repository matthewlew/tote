## 2024-04-25 - Global Escape Navigation & Focus Restoration
**Learning:** Native single-page-app transitions that manage multiple overlays (neighborhood picker, collection detail, expanded item) often lack robust keyboard accessibility. Furthermore, structural characters (arrows, shortcut spans) clutter screen reader experiences if not explicitly hidden.
**Action:** Always implement a history stack or track the opening trigger element to consistently restore focus (`element.focus()`) upon dismissing views via `Escape` or "Back" buttons. Hide all structural iconography from screen readers (`aria-hidden="true"`) when semantic meaning is already clear.
