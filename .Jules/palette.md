## 2024-05-18 - Implement explicit Escape key navigation
**Learning:** For a deep navigation structure (e.g., location -> main app -> detail screens -> expanded rows), a single hardcoded "back" action or screen ID mapped to the 'Escape' key is insufficient. Implementing a real stack (`screenHistory`) provides robust back-tracking.
**Action:** When adding global 'Escape' key dismissals, always manage state with a LIFO stack for deep navigation and prioritize dismissing specific inline overlays (like expanded list items) before executing screen-level back navigation. Always provide an explicit `<span class="shortcut">esc</span>` visual hint for the user.
